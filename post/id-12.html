Cloudflare Pages 体验评价

2021年5月4日 由 秋月夏茗

本文长时间未更新，部分内容可能已过时。

Pages 服务，即静态页面托管服务，是用于托管静态网页的一种便捷途径，最出名的有 GitHub Pages、Vercel Pages、Gitee Pages 等。最近，Cloudflare 也推出了 Pages 服务，下面是关于此服务的体验评价。

若要使用 Cloudflare Pages，首先需注册 Cloudflare，接着在控制台主页点击进入“网页”链接即可。

Cloudflare Pages 不提供文件托管，目前只支持连接 GitHub 仓库获取网页源文件。

部署过程和 Vercel 几乎一致，而且这个浮夸的首次部署成功效果也如此相仿。

部署控制系统还是非常完善的，每次部署都有单独的链接，方便开发使用。

Cloudflare Pages 成功部署后会分配一个默认二级域名，也支持自行绑定域名。

将自有域名输入至 Cloudflare 并自行添加 CNAME 记录即可，不强制通过 NS 接入。

添加自定义域名后需等待 Cloudflare 进行解析添加校验并签发 SSL 证书，否则自定义域名无法使用。

测试时分配的两个 IP 分别为北美104.21.16.226和欧洲172.67.216.70。境内直接访问时，就要在并不普遍适合三大运营商的洛杉矶和玄学的欧洲 Level3 之间做命运大抽奖。

站长之家测得全球 Ping 延时比以 NS 方式接入的 Cloudflare CDN 稍高。

站长之家测得的境内访问总时间（排除域名解析时间），效果不理想。

综上，Cloudflare Pages 具备基础的页面部署、版本分离、自定义域名功能，但默认分配的线路质量一般，不适合中国境内使用（Cloudflare 只通过 DNS 校验域名是否绑定，因此无法自选节点或在用户与 Cloudflare Pages 之间再套 CDN 等），仍需等待进一步完善。
